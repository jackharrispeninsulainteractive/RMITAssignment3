Aim Overview

Crew us is an app designed to put gamers in contact. Its main aim is to allow the players to find someone to play with and organise their gaming session through a messaging system. To achieve this goal, there are smaller steps that we need to take first.


Aims Breakdown

  1. The first step is to create Accounts with relevant Log In/Registration interface. Every user will need to have an account to message another user. At first the users will need to create an account, and for the future access they will just need to insert username and password. The reason why the creation of an account and of a secure login interface is the first step because all the subsequent functions will not work without this step. The accounts will then have a profile page where information like username and the games played by the user will be displayed. Our job is to create profile pages for users that allow them to insert their information in a simple way. This page needs to be graphically appealing, but most of all it needs to be easy to read, so that users can understand straight away if they are looking at someone they can play with or not.

  2. The second step is to develop the finding function. This allows the user to find other players, so that they can message them or open their profile. Probably at the beginning it will be possible to search only by username, but the future aim is to enable users to find other players by game. This function is obviously necessary and without it the whole aim of our app would not be reached. The gamer would not be able to do anything except the creation of their profile.

  3. The last step is developing the messaging system. Despite the fact that it is almost the main function, it is the last goal to achieve since it would not be usable before the previous steps. The messaging system will allow the gamers to send messages and media like images to each other. We are aiming to create private chat rooms where two players can write to each other and Crew rooms, which will include more than two members. So it is necessary to create a reserved space which will include all the messages of a certain chat room, a text box that allows the user to write the next message, a media button which will allow the player to select a file and a sending button to send the message typed in the box.
