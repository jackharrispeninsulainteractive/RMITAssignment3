Trello - Project Management

Trello was used more than MS Teams for project management as it allows us to assign users to tasks and create check lists based on the assigment criteria. An in-depth look at who performed what task can be seen on trello by seeing what users are assigned to what items.


Trello Reflection

The decision to use trello for our group was a fantastic one, whilst MS Teams is great for communications trello is fantastic for project management, its really allowed us to stay on top of all the tasks and provide a quick and easy system for converting the large assessment criteria into manageable checklists to complete. All the group members have used trello during this assessment, some feedback to Alex, Noah and Jaydyn would be to tick of individual chunks of the assessment as you complete them rather than leaving it all to the end before ticking it all off.



Github Commit Overview

Looking back at our GitHub commit trail we can see that everyone who submitted content has made an effort to include appropriate logs to identity the changes that were made to the files, For our group Jack performed a lot of the site work and Alex, Noah And Jaydyn sent content to him before uploading, this was done to prevent major branch deviations and streamline the assignment.


GitHub Reflection

Look at the comments from Jack & Alex here we can clearly see that they have displayed exactly what was worked on,With Jacks we have to display the full comment has his is dot pointed. At the start Noah created and uploaded the first copy of the site, he didnt include any comments for this so some fantastic feedback for next time would be to let him know to comment everything, even the first additions

MS Teams - Team Communication

During this assessment we have been using Microsoft Teams for all our group communication & Meetings, this has given us some great experience with the app, we have compiled a reflection below. Teams has a great integrated meeting system with online recording that is fantastic but over all as a group our workflow didn’t function too well with MS Teams.

Each group member made a fantastic effort to use it as our point of communication, Each group member used a webcam when required, we did find that with some slow connections it was best to turn them off whilst screensharing.


MS Teams Reflection

After using teams for a number of weeks during this project our group has come to the conclusion that we do not like the work flow or feature set of teams offers, We do not really believe it provided a lot of benefit, looking at all the tools around and having used Slack for a RMIT short course and work before we think it would have been a better option when combined with a tool like Trello.

Microsoft want to break into this market, but Teams is trying to be too many things and as a result failing at all of them, it provides no benefit for project management, its chat system is clunky and segregated. The main positive was the integrated recordable video, this is a great feature but overall, our group did not like using teams for project management & communication and we feel there are better tools to suit or workflow.
